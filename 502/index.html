Suppose LeetCode will start its IPO soon. In order to sell a good price of its shares to Venture Capital, LeetCode would like to work on some projects to increase its capital before the IPO. Since it has limited resources, it can only finish at most k distinct projects before the IPO. Help LeetCode design the best way to maximize its total capital after finishing at most k distinct projects.

You are given n projects where the ith project has a pure profit profits[i] and a minimum capital of capital[i] is needed to start it.

Initially, you have w capital. When you finish a project, you will obtain its pure profit and the profit will be added to your total capital.

Pick a list of at most k distinct projects from given projects to maximize your final capital, and return the final maximized capital.

The answer is guaranteed to fit in a 32-bit signed integer.

Example 1:

Input: k = 2, w = 0, profits = [1,2,3], capital = [0,1,1] Output: 4 Explanation: Since your initial capital is 0, you can only start the project indexed 0. After finishing it you will obtain profit 1 and your capital becomes 1. With capital 1, you can either start the project indexed 1 or the project indexed 2. Since you can choose at most 2 projects, you need to finish the project indexed 2 to get the maximum capital. Therefore, output the final maximized capital, which is 0 + 1 + 3 = 4.

Example 2:

Input: k = 3, w = 0, profits = [1,2,3], capital = [0,1,2] Output: 6

Constraints:

  • 1 <= k <= 105
  • 0 <= w <= 109
  • n == profits.length
  • n == capital.length
  • 1 <= n <= 105
  • 0 <= profits[i] <= 104
  • 0 <= capital[i] <= 109

pub fn find_maximized_capital(k: i32, w: i32, profits: Vec, capital: Vec) -> i32 {
    let mut max_capital = w;
    let mut projects = vec![0; k as usize];
    let mut i = 0;
    while i < k as usize {
        let mut min_capital = i32::MAX;
        let mut min_index = -1;
        for j in 0..capital.len() {
            if projects[j as usize] == 0 && capital[j as usize] < min_capital {
                min_capital = capital[j as usize];
                min_index = j as i32;
            }
        }
        if min_index != -1 {
            projects[min_index as usize] = 1;
            max_capital += profits[min_index as usize];
        } else {
            break;
        }
        i += 1;
    }
    max_capital

impl Solution {
    pub fn find_maximized_capital(k: i32, w: i32, profits: Vec, capital: Vec) -> i32 {
        let mut max_capital = 0;
        let mut current_capital = w;
        let mut projects = Vec::new();

        for i in 0..profits.len() {
            if current_capital >= capital[i] {
                projects.push((profits[i], capital[i]));
                current_capital -= capital[i];
            }
        }

        projects.sort_by(|a, b| b.0.cmp(&a.0));

        for i in 0..k {
            if i < projects.len() {
                max_capital += projects[i].0;
                current_capital += projects[i].1;
            }
        }

        max_capital + current_capital
    }
}
